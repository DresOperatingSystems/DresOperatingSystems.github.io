Magisk Modules Live

We are building Magisk modules to automate the DresOS Android build process step by step. Each module replaces a block of manual steps with a single flash in Magisk. The goal is a complete automated DresOS build.

These modules bypass many of the restrictions Google has put in place while continuing the fight against data mining, data selling, and surveillance. Due to those restrictions we shut down our app projects including DresSecureComms - but we are not going anywhere. We redirected everything into this.

Released Modules

dresoswv v1.0.0
DresOS AOSmium WebView
Replaces Android's system WebView with AOSmium - a Chromium fork hardened with GrapheneOS/Vanadium security patches. The system WebView is the browser engine used by hundreds of apps whenever they render web content. Google's version phones home on every render. This module replaces it in a single Magisk flash.

What it does: hides all competing WebView packages, installs AOSmium to system/app, patches the config_webview_packages allowlist via RRO overlay, registers via pm install, supports Magisk and KernelSU.

After flashing: Settings -> Developer Options -> WebView implementation -> select AOSmium WebView.

DresOS Android

We originally wanted to release our own system but due to all these new cyber laws and the pressure Google is putting on firmware swaps we just could never release it. So we hope our project can bring our organisation back to its core values and release an idea we had once in a different light.

The DresOS Android project has been fully merged into a single comprehensive guide. The old two-repo method (Android-Degoogling + Android-opsec) is now combined into one unified system with a full app suite, security architecture, and changelog.

Build Your DresOS Android System

Follow the unified guide below. It covers everything from unlocking the bootloader and flashing Magisk, to the full OPSEC stack with InviZible Pro, AFWall+, and the complete DresOS defensive app suite.

Full Guide Security Architecture Changelog

What the System Includes

  • Part 1 - DeGoogling - Full root method: TWRP, Magisk, Noogle microG, AOSmium WebView via Magisk module, Shizuku, system debloat, FOSS app replacements
  • Part 2 - Operational Security - InviZible Pro proxy+root mode (Tor + I2P + DNSCrypt), AFWall+ kernel firewall, Tuta Mail + Duck Address email setup, GPS spoofing
  • Part 3 - DresOS Defensive App Suite - 16 apps fully documented with setup, features, and usage

How this system protects you

  • Removes all Google services and tracking
  • Replaces system WebView with AOSmium via Magisk module (GrapheneOS hardened)
  • Routes all traffic through Tor via InviZible Pro proxy+root mode
  • Encrypts all DNS queries via DNSCrypt at kernel level using iptables
  • Blocks in-app trackers via DuckDuckGo App Tracking Protection
  • Kernel-level firewall via AFWall+ with per-app rules
  • AES-256 file encryption via Amaze File Manager
  • Scans every link before opening via URL Check
  • Scans every APK before installing via Hypatia (ClamAV)
  • End-to-end encrypted email via Tuta Mail with Duck Address aliases
  • GPS spoofing via Fake Traveler
  • MAC address randomisation per network
